Teamwork OAuth

Create App

1. Go to developer.teamwork.com and create an OAuth app

2. Set the Redirect URI to:

{{ base_url }}/oauth/teamwork/callback

3. Copy the Client ID and Client Secret below

Slack OAuth

Create App

1. Go to api.slack.com/apps and create a new app

2. Under OAuth & Permissions, add the Redirect URL:

{{ base_url }}/oauth/slack/callback

3. Add the required User Token Scopes listed below

4. Copy Client ID and Client Secret from Basic Information

Required User Token Scopes (16)
channels:readList public channels and their info
channels:historyRead message history in public channels
groups:readList private channels the user is in
groups:historyRead message history in private channels
chat:writeSend messages on behalf of the user
im:readList direct message conversations
im:writeStart new direct message conversations
im:historyRead message history in DMs
mpim:readList group DM conversations
mpim:writeStart new group DM conversations
mpim:historyRead message history in group DMs
users:readView user profiles and presence
users:read.emailView user email addresses
search:read.publicSearch messages in public channels
search:read.filesSearch shared files
canvases:readRead Slack canvases
canvases:writeCreate and edit Slack canvases

Miro OAuth

Create App

1. Go to Miro Developer Settings and create a new app

2. In the app settings, add the Redirect URI:

{{ base_url }}/oauth/miro/callback

3. Enable the required scopes listed below

4. Copy Client ID and Client Secret

Required Scopes (4)
boards:readView boards, items, and their content
boards:writeCreate and edit boards, sticky notes, shapes, cards, connectors
identity:readIdentify the connected user (name, ID)
team:readView team info and membership

Figma OAuth

Create App

1. Go to Figma Developer Apps and create a new app

2. Add the Callback URL:

{{ base_url }}/oauth/figma/callback

3. Enable the required scopes listed below

4. Copy Client ID and Client Secret

Required Scopes (5)
file_content:readRead layers, nodes, and design content of files
file_metadata:readRead file name, versions, thumbnail, and project info
file_comments:writePost and delete comments on files
file_dev_resources:readRead dev resources and component links
current_user:readIdentify the connected user (name, email, avatar)

Telegram (MTProto)

Create App

1. Go to my.telegram.org/apps and sign in

2. Create a new application (or use an existing one)

3. Copy the API ID (numeric) and API Hash

No redirect URI needed — Telegram uses session-based authentication
